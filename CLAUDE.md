```
src/
├── app/
│   ├── layout.tsx
│   ├── page.tsx
│   └── [slug]/page.tsx
├── components/
│   ├── layout/

│   │   └── Footer.tsx
│   ├── sections/
│   │   ├── HeroSection.tsx
│   │   ├── EcosystemSection.tsx

│   │   ├── KeyFeatures.tsx
│   │   ├── HowItWorks.tsx
│   │   ├── PricingSection.tsx
│   │   ├── Testimonials.tsx
│   │   ├── TrustedBy.tsx
│   │   ├── FAQSection.tsx
│   │   └── CTABanner.tsx
│   └── ui/
├── lib/utils.ts
└── data/
    ├── services.ts
    ├── pricing.ts
    ├── testimonials.ts
    └── faq.ts
```

### Rules
- Components: PascalCase, one per file
- Tailwind only — no inline `style={}` except dynamic CSS variables
- No `any` in TypeScript
- Vietnamese text lives in `data/` files, imported into components
- Images: `next/image` with `alt`
- Animations: Framer Motion `variants`, respect `prefers-reduced-motion`

### Accessibility
- Semantic HTML: `<header>`, `<main>`, `<section>`, `<footer>`, `<nav>`
- Keyboard-navigable interactive elements
- Contrast ratio: 4.5:1 body, 3:1 large text
- `aria-label` on icon-only buttons

## 14. Quy tắc bắt buộc (Mandatory Rules)

> Các quy tắc này áp dụng cho **mọi thay đổi** trong suốt dự án, không có ngoại lệ.

### R1 — Screenshot & So sánh sau mỗi thay đổi lớn
- Sau mỗi section hoàn thành hoặc thay đổi layout đáng kể, chụp screenshot trang web đang chạy (`npm run dev`)
- So sánh trực tiếp với ảnh design gốc (`anh web mau 1.jpg`) về: màu sắc, spacing, typography, card layout
- Nếu có sai lệch rõ ràng → sửa trước khi chuyển sang task tiếp theo
- Không báo task hoàn thành nếu chưa qua bước kiểm tra visual này

### R2 — Tối ưu Mobile bắt buộc
- Mọi component phải được test ở breakpoint: `375px` (mobile), `768px` (tablet), `1280px` (desktop)
- Không được có overflow ngang (`overflow-x: hidden` trên body là không đủ — phải fix tận gốc)
- Navigation: hamburger menu trên mobile, full nav trên desktop (`md:`)
- Font size tối thiểu: `16px` trên mobile để tránh auto-zoom iOS
- Touch target tối thiểu: `44x44px` cho tất cả button và link
- Spacing mobile: giảm section padding xuống `py-12` thay vì `py-20` trên `sm:`
- Grid: mặc định `grid-cols-1` mobile → `grid-cols-2` tablet → `grid-cols-3/4` desktop

### R3 — Animation bắt buộc cho mọi action
Mọi element tương tác đều phải có visual feedback khi click/tap:

**Buttons:**
```tsx
// Pattern chuẩn cho tất cả button
whileTap={{ scale: 0.96 }}
whileHover={{ scale: 1.02 }}
transition={{ type: "spring", stiffness: 400, damping: 17 }}
```

**Cards (clickable):**
```tsx
whileHover={{ y: -4, boxShadow: "0 20px 40px rgba(255,107,0,0.15)" }}
whileTap={{ scale: 0.98 }}
transition={{ duration: 0.2 }}
```

**Links / Nav items:**
```tsx
// Underline slide-in hoặc color transition
className="transition-colors duration-200 hover:text-[#FF6B00]"
```

**Scroll-triggered sections:**
```tsx
// Mọi section dùng fade-up khi vào viewport
initial={{ opacity: 0, y: 30 }}
whileInView={{ opacity: 1, y: 0 }}
viewport={{ once: true, margin: "-80px" }}
transition={{ duration: 0.5, ease: "easeOut" }}
```

**Form inputs / Interactive elements:**
```tsx
// Focus ring màu cam
className="focus:ring-2 focus:ring-[#FF6B00] focus:outline-none transition-all duration-200"
```

> `prefers-reduced-motion`: bọc tất cả animation bằng `useReducedMotion()` hook — nếu user tắt animation thì fallback về `opacity` transition đơn giản.

## 15. Design Principles

1. **Dark-first** — dark sections dominate; light for contrast
2. **Orange = action** — reserved for CTAs and key highlights only
3. **Data credibility** — show numbers, stats, social proof prominently
4. **Generous whitespace** — never crowd; let content breathe
5. **Consistent motion** — subtle fade-up on scroll, no jarring transitions
6. **Mobile-first** — design for mobile, enhance for desktop
7. **Vietnamese-first** — content, font, and UX optimized for Vietnamese users
